Generation method:
  1000 students × top 100 jobs per student = 1,00,000 rows
  
  How to pick top 100:
    For each student, compute cosine similarity with all 500 jobs
    Sort descending by similarity
    Keep top 100 only
    This ensures we have the most relevant pairs

COLUMN 1: match_id
  Format:  M000001 to M100000

COLUMN 2: student_id
  FK → student_profiles.student_id

COLUMN 3: job_id
  FK → job_postings.job_id

COLUMN 4: semantic_score
  Type:    Float, 4 decimal places, 0.0 to 1.0
  
  How to compute:
    model = SentenceTransformer("all-MiniLM-L6-v2")
    s_vec = model.encode(student.resume_text_raw)
    j_vec = model.encode(job.description_raw)
    score = cosine_similarity([s_vec], [j_vec])[0][0]
    score = max(0.0, min(1.0, float(score)))
  
  Expected range in real data: 0.35 to 0.95
  (Perfect 1.0 never happens, below 0.3 means no relevance)

COLUMN 5: academic_score
  Type:    Float, 4 decimal places, 0.0 to 1.0
  
  Formula:
    IF job.min_gpa == 0.0:
        academic_score = 1.0
    ELSE:
        raw = (student.gpa - job.min_gpa) / (10 - job.min_gpa)
        academic_score = max(0.0, min(1.0, raw))

COLUMN 6: preference_score
  Type:    Float, 4 decimal places, 0.0 to 1.0
  
  Formula (average of 4 sub-scores now — more comprehensive):
  
    domain_match:
      IF student.preferred_domain == job.domain → 1.0
      IF student.preferred_domain == "Any"      → 1.0
      ELSE                                       → 0.0
    
    location_match:
      IF job.work_type == "Remote" → 1.0
      IF job.location in student.preferred_locations → 1.0
      IF "Remote" in student.preferred_locations
         AND job.work_type in ("Remote","Hybrid")  → 0.8
      ELSE → 0.0
    
    skill_overlap:
      required = set of job.required_skills
      student  = set of student.skills
      overlap  = len(required ∩ student)
      skill_overlap = overlap / len(required)
    
    role_match:
      IF any(student.preferred_roles overlaps job.role_title) → 1.0
      ELSE → 0.0
    
    preference_score = (domain + location + skill_overlap + role_match) / 4

COLUMN 7: experience_score
  Type:    Float, 4 decimal places, 0.0 to 1.0
  Formula: min(student.experience_months / job.required_experience_months, 1.0)
           IF job.required_experience_months == 0: score = 1.0

COLUMN 8: skill_gap_score
  Type:    Float, 4 decimal places, 0.0 to 1.0
  NEW COLUMN — more informative than simple overlap
  
  Formula:
    required = set(job.required_skills)
    student  = set(student.skills)
    missing  = required - student
    skill_gap_score = 1 - (len(missing) / len(required))
    
  Example:
    Job requires: Python|SQL|Pandas|Docker (4 skills)
    Student has:  Python|SQL|Pandas (3 skills)
    Missing:      Docker (1 skill)
    skill_gap_score = 1 - (1/4) = 0.75
  
  Purpose: SHAP can explain "you are missing Docker"

COLUMN 9: certification_bonus
  Type:    Float, 0.0 or 0.1 or 0.2
  
  Formula:
    Checks if student certifications match job domain
    0.0 = no relevant certification
    0.1 = 1 relevant certification
    0.2 = 2+ relevant certifications
  
  Example:
    Job domain: Data Science
    Student cert: "Coursera Machine Learning by Andrew Ng"
    → certification_bonus = 0.1

COLUMN 10: github_activity_score
  Type:    Float, 4 decimal places, 0.0 to 1.0
  Formula: min(student.github_projects_count / 20, 1.0)
  Purpose: Engagement signal for technical roles

COLUMN 11: backlog_penalty
  Type:    Float, 0.0 or -0.1 or -0.2
  
  Formula:
    IF job.backlog_allowed == True: backlog_penalty = 0.0
    IF student.backlogs == 0: backlog_penalty = 0.0
    IF student.backlogs == 1 AND backlog_allowed==False: -0.1
    IF student.backlogs >= 2 AND backlog_allowed==False: -0.2
  
  Purpose: Hard filter — some companies reject backlogs

COLUMN 12: hybrid_score
  Type:    Float, 4 decimal places, 0.0 to 1.0
  
  THIS IS THE FINAL MATCH SCORE SHOWN TO USER
  
  Formula:
    base = (0.45 × semantic_score)
         + (0.25 × academic_score)
         + (0.20 × preference_score)
         + (0.10 × skill_gap_score)
    
    final = base + certification_bonus + backlog_penalty
    hybrid_score = max(0.0, min(1.0, final))
  
  WHY THESE WEIGHTS:
    Semantic (45%) — most important, captures overall fit
    Academic (25%) — GPA matters but not everything
    Preference (20%) — location, domain, role match
    Skill gap (10%) — how many specific skills they have

COLUMN 13: fairness_corrected_score
  Type:    Float, 4 decimal places, 0.0 to 1.0
  
  This is hybrid_score AFTER fairness correction.
  
  The correction normalizes scores within college_tier groups
  so that a tier-3 student with same skills as a tier-1 student
  gets a fair ranking.
  
  Formula (applied AFTER generating all hybrid_scores):
    For each tier group (1, 2, 3):
      group_scores = all hybrid_scores for that tier
      mean = average of group_scores
      std  = standard deviation
      
      corrected = (raw_score - mean) / std × 0.12 + mean
      corrected = max(0.0, min(1.0, corrected))
  
  Purpose: This is what the app actually uses for ranking.
           Not hybrid_score directly.

COLUMN 14: was_selected
  Type:    Integer, 0 or 1
  TARGET variable for ML training
  
  Probability assignment:
    fairness_corrected_score > 0.80 → 85% chance = 1
    0.65 to 0.80                    → 45% chance = 1
    0.50 to 0.65                    → 20% chance = 1
    0.35 to 0.50                    →  8% chance = 1
    below 0.35                      →  2% chance = 1
  
  Add random noise: 3% flip (1→0 or 0→1) randomly
  
  Why noise: Real hiring has randomness.
  Great candidates get rejected. Average ones get lucky.
  Model must learn this uncertainty.

COLUMN 15: rank_in_job
  Type:    Integer
  
  For each job, rank all students by fairness_corrected_score.
  Student with highest score gets rank 1.
  
  Example: Job J0015 has 120 applicants.
           Student S0042 scores 0.87 → rank 3

COLUMN 16: college_tier
  Copied from student_profiles.
  Used ONLY for fairness analysis.
  NOT a model feature.
