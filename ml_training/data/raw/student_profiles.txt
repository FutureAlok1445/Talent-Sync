COLUMN 22: resume_text_raw  ← MOST CRITICAL
  Type:    Long string, natural English paragraph
  Length:  100 to 200 words per student
  
  THIS IS WHAT SBERT READS AND ENCODES.
  QUALITY HERE = QUALITY OF ENTIRE MATCHING SYSTEM.
  
  What to include:
    → Degree, branch, college name
    → GPA (mention it naturally)
    → All skills (mentioned naturally, not listed robotically)
    → Project descriptions with technologies
    → Experience if any
    → Certifications if any
    → Achievements if any
    → What roles they want
    → Location preference
    → Domain interest
  
  3 different writing styles (mix across 1000 students):
  
  STYLE 1 — Professional summary (350 students):
    "Final year B.Tech Computer Science student from VJTI Mumbai
     maintaining a CGPA of 8.5. Proficient in Python, React, SQL,
     and Pandas with hands-on experience building data-driven web
     applications. Developed a sentiment analysis tool using BERT
     that achieved 92% accuracy, and an e-commerce platform handling
     real-time inventory updates. Completed Google Data Analytics
     Certificate and contributed to 8 open-source GitHub projects.
     3 months of internship experience in backend development at a
     SaaS startup. Actively seeking Data Analyst or ML Engineer
     roles in Mumbai or Pune, with interest in FinTech domain."
  
  STYLE 2 — Skills-focused (350 students):
    "I am a Data Science student from NIT Surat with 8.8 GPA and
     strong foundations in machine learning, deep learning, and
     natural language processing. My technical toolkit includes
     Python, TensorFlow, PyTorch, Scikit-learn, and SQL. I have
     built predictive models for stock price forecasting using LSTM
     networks and worked on a fraud detection system using ensemble
     methods. Ranked in top 5% on LeetCode with 1200+ rating.
     Looking for ML Engineer or Data Scientist internship roles in
     Bangalore or Remote. Preferably in HealthTech or SaaS."
  
  STYLE 3 — Project-focused (300 students):
    "Engineering student specializing in full-stack web development.
     Core skills: JavaScript, React, Node.js, MongoDB, and REST API
     design. Built three production-ready projects including a food
     delivery app with real-time tracking, a collaborative task
     manager with WebSocket, and a portfolio generator tool. Familiar
     with Docker for containerization and Git for version control.
     Completed Meta Frontend Developer Certificate on Coursera.
     Open to Frontend Developer and Full Stack roles anywhere in
     India or Remote. Interested in EdTech and E-commerce domains."
  
  CRITICAL RULES:
    1. Never copy-paste same text for multiple students
    2. Use synonyms: "proficient in" / "skilled in" / "experienced with"
       "built" / "developed" / "created" / "designed" / "implemented"
    3. Include numbers where possible: "92% accuracy", "8 GitHub projects"
       "handled 10,000 requests", "3 months experience"
    4. SBERT understands: "data manipulation" = "Pandas"
       Write naturally — do not just list skill names
